# Slack PR Summary Command

You generate a compact Slack-formatted message summarizing GitHub PRs. The user provides PR URLs as `$ARGUMENTS`.

## Instructions

1. **Parse PR URLs** from `$ARGUMENTS` (space-separated GitHub PR URLs)
2. **For each PR**, run `gh pr view <url> --json url,headRepository,headRepositoryOwner,additions,deletions` to extract:
   - PR URL
   - Additions and deletions count
   - Repository language (run `gh api repos/{owner}/{repo}/languages` to get the primary language)
3. **Map language to Slack emoji** using the reference table below
4. **Analyze all PRs** to generate a short, natural intro sentence describing what these PRs are about (e.g. common theme, affected area, release context). Keep it to 1 sentence max.
5. **Output the final Slack message** in this exact format:

```
Hello everyone 👋

<auto-generated intro sentence>

<emoji> - <url> (+additions -deletions)
<emoji> - <url> (+additions -deletions)
...
```

## Language → Slack Emoji Mapping

| Language      | Slack Emoji  |
| ------------- | ------------ |
| Ruby          | :ruby:       |
| Go            | :golang:     |
| TypeScript    | :typescript: |
| JavaScript    | :javascript: |
| Python        | :python:     |
| Rust          | :rust:       |
| Java          | :java:       |
| Kotlin        | :kotlin:     |
| Swift         | :swift:      |
| C#            | :csharp:     |
| C++           | :cpp:        |
| PHP           | :php:        |
| Elixir        | :elixir:     |
| Dart          | :dart:       |
| Shell/Bash    | :bash:       |
| HTML/CSS      | :html:       |
| Dockerfile    | :docker:     |
| Terraform     | :terraform:  |
| Unknown/Mixed | :github:     |

Use the language with the most bytes from the languages API response as the primary language.

## Behavior

- **No confirmation needed** — just output the message
- **Keep it short** — this is meant to be pasted directly into Slack
- **Plain text links** — output bare URLs (Slack's composer auto-links them). Do NOT use `<url|title>` mrkdwn syntax or `[title](url)` markdown — neither works when pasting into Slack's rich text editor
- **Group by repo** if PRs come from multiple repos — add a blank line between groups
- **If a PR URL is invalid or inaccessible**, skip it and note at the end: `⚠️ Could not access: <url>`

## Output Example

```
Hello everyone 👋

Here are the latest PRs improving our authentication flow and fixing webhook handling.

:ruby: - https://github.com/org/rails-app/pull/42 (+45 -12)
:ruby: - https://github.com/org/rails-app/pull/43 (+123 -22)
:golang: - https://github.com/org/go-service/pull/15 (+67 -8)

Could you have a look, please? Thanks! 🫶
```

## Important

- Output ONLY the Slack message — no explanations, no markdown code blocks, no extra commentary
- The message should be ready to paste into Slack as-is
- After outputting the message, pipe it to `pbcopy` so it's automatically copied to the clipboard. Use `echo -e "<message>" | pbcopy` and confirm to the user it's been copied.
